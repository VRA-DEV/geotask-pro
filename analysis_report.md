## Mudança Necessária (Nova feature)
Para suportar a hierarquia solicitada (`Contrato -> Cidade -> Bairro`) sendo que algumas cidades aparecem em mais de um contrato, mas os bairros mudam, precisamos relacionar `Neighborhood` (Bairro) ao `Contract` (Contrato) no banco de dados.

### Plano de Ação:
1. **Schema Prisma**: Adicionar no schema do banco que `Neighborhood` tenha `contract_id`.
2. **Migração do Banco**: Rodar comando de db push para que o banco passe a ter o ID do Contrato junto ao Bairro.
3. **Script de Seed Segura**: Criar um seed do zero puxando as relações precisas desse Excel e preenchendo o BD sem deletar as tarefas.
4. **Frontend/Modais de Tarefas**: Implementar regras para que a City apareça filtrada pelo Contrato, e Bairro filtrado pela Cidade+Contrato.
